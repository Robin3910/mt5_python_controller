## 1. 系统简介

一套「中心管理端 + 多 MT5 节点」的信号跟单系统：

- **管理端（后端 + Web）**：接收 TradingView Webhook 信号，按规则分发给各 MT5 节点；提供节点管理、手数/过滤/分发配置、账户监控、远程平仓。
- **节点端（MT5 Node）**：运行在装有 MT5 终端的 Windows 主机上，通过 WebSocket 长连接接收命令并执行下单/平仓，同时上报账户与持仓。

```
TradingView ──HTTP Webhook──▶ 后端(FastAPI)
                                  │  ├─ Redis（实时态/队列/锁）
                                  │  └─ MySQL（节点账本/审计/历史）
                                  │
                      ┌───────────┼───────────┐  WebSocket(wss)
                      ▼           ▼           ▼
                  MT5 节点A    MT5 节点B    MT5 节点C
                      ▲
   管理员 ──Web(Vue)──┘ （登录后通过 /ws/admin 实时看板 + 远程平仓）
```

技术栈：FastAPI + Redis + MySQL（后端），Vue3 + TypeScript（前端），`websockets` + MetaTrader5（节点端）。

## 4. 快速开始（本地开发，免 Docker / 免 MySQL / 免真实 MT5）

适合先把全链路跑通、做联调。后端用 SQLite + 本地 Redis，节点端用模拟 MT5。

### 4.1 启动 Redis

```bash
# 任选其一
docker run -d --name redis -p 6379:6379 redis:7-alpine
# 或本机已安装的 redis-server
```

### 4.2 启动后端

```bash
cd backend
python -m venv .venv && .venv\Scripts\activate     # Windows
# source .venv/bin/activate                          # macOS/Linux
pip install -r requirements.txt
copy .env.example .env                               # Windows；Linux 用 cp
# .env 默认即为：SQLite + 本地 Redis + 关闭鉴权，适合本地
uvicorn app.main:app --reload --port 8000
```

启动成功后：

- API 文档：<http://localhost:8000/docs>
- 健康检查：<http://localhost:8000/health>

### 4.3 启动前端

```bash
cd frontend
npm install
npm run dev        # 默认 http://localhost:5173，已配置代理到 :8000
```

浏览器打开 <http://localhost:5173>，用 `.env` 中的 `ADMIN_USER/ADMIN_PASSWORD`（默认 `admin / admin123`）登录。

### 4.4 拷贝全局令牌并启动「模拟」节点端

1. Web → 「配置」→「节点令牌 (NODE_TOKEN)」→ 点击「显示」并复制（首次启动后端时已自动生成）。
2. 配置并启动节点端（模拟模式）：

```bash
cd node_client
python -m venv .venv && .venv\Scripts\activate
pip install -r requirements.txt
copy .env.example .env
# 编辑 .env：把 NODE_TOKEN 设为上一步复制的全局令牌；设 MT5_LOGIN=<任意正整数> 和 MT5_MOCK=true
python node_client.py
```

3. 节点首次连接时，后台按 `MT5_LOGIN` 在库中查找；若不存在则**按默认配置自动入库**（名称 `node-<login>`、**禁用**；**按币种配置**从中控台已登记品种自动生成，每个品种默认参与 sync/poll、固定手数 0.01、轮询序 0）。本轮鉴权会因禁用被拒（`4403`），需在「节点管理」启用后才会上线。
4. 在 Web 「节点管理」将该节点设为**启用**后，重启或等待 node_client 重连；回到「总览」应能看到该节点 **上线**、并周期上报模拟账户。

> 也可以反过来：先在「节点管理 → + 新建节点」里手动创建（填入 mt5_login；手动新建默认启用），再启动 node_client 接入。

### 4.5 在中控台登记信号品种

Webhook 只会处理**已在中控台登记**的品种；未登记的信号会返回 `status: rejected`（HTTP 200，但不分发）。

1. Web → 「中控台」→「多区间方向过滤」→ 点击「+ 添加品种」，填入 `XAUUSD`（或你测试用的品种）。
2. 保持默认即可（分发模式 `sync`、持仓范围 `symbol`、允许做多/做空），点击「保存过滤规则」。

### 4.6 触发一次信号

```bash
curl -X POST http://localhost:8000/webhook ^
  -H "Content-Type: application/json" ^
  -d "{\"action\":\"buy\",\"symbol\":\"XAUUSD\",\"volume\":0.1}"
```

响应应含 `"status":"accepted"`。总览页事件流会出现「分发 / 回报」记录，节点持仓 +1。至此本地全链路打通。

## 5. 生产部署（Docker Compose 一键）

服务端一条命令拉起 `mysql + redis + backend + nginx`，前端以静态资源由 Nginx 托管。

### 5.1 准备环境变量

```bash
cp .env.example .env
```

按 [§7 配置说明](#7-配置项说明) 修改 `.env`，**务必修改以下敏感项**：

```
MYSQL_ROOT_PASSWORD / MYSQL_PASSWORD   # 数据库密码
JWT_SECRET                              # 足够长的随机串
ADMIN_PASSWORD                          # 管理后台密码
ENABLE_AUTH=true / AUTH_TOKEN           # Webhook 鉴权
```

### 5.2 先构建前端静态资源

Nginx 通过挂载 `frontend/dist` 来托管前端，因此需先构建：

```bash
cd frontend
npm install
npm run build      # 产物输出到 frontend/dist
cd ..
```

### 5.3 启动全部服务

```bash
docker compose up -d --build
docker compose ps          # 确认 mysql/redis healthy，backend/nginx up
docker compose logs -f backend
```

访问入口（默认 80 端口，由 Nginx 统一转发）：

| 路径 | 说明 |
| --- | --- |
| `http://<服务器IP>/` | 管理后台（Web） |
| `http://<服务器IP>/webhook` | TradingView Webhook |
| `http://<服务器IP>/docs` | 后端 API 文档 |
| `http://<服务器IP>/health` | 健康检查 |
| `ws(s)://<服务器IP>/ws/node` | 节点接入 |
| `ws(s)://<服务器IP>/ws/admin` | 后台实时推送 |

### 5.4 数据库迁移（生产）

容器内已包含 Alembic。首次或升级时执行：

```bash
docker compose exec backend alembic upgrade head
```

> 说明：后端启动时也会自动 `create_all` 建表（便于快速启动）；生产环境**推荐以 Alembic 迁移为准**进行版本化管理。

### 5.5 启用 HTTPS（强烈建议）

1. 把证书放到 `deploy/nginx/certs/`（`fullchain.pem`、`privkey.pem`）。
2. 打开 `deploy/nginx/default.conf` 末尾被注释的 `443` server 段，填好 `server_name`。
3. `docker compose restart nginx`。
4. 节点端与 TradingView 改用 `wss://` / `https://`。

### 5.6 Windows 服务器部署（Docker Desktop）

仓库额外提供 `docker-compose.windows.yml`，与上面的 Linux 版**拓扑完全一致**（mysql + redis + backend + nginx 全在容器内，节点端在各自 Windows 主机原生运行）。容器内部都是 Linux 镜像，宿主是 Windows 不影响其行为；唯一面向 Windows 的差异是把 nginx 对外端口参数化——默认 `8080/8443`，绕开 Windows 上常被 IIS / HTTP.sys 占用的 `80/443`。

前置条件：

1. 安装 Docker Desktop（建议 WSL2 后端）。
2. 仍需先构建前端（§5.2）：`cd frontend; npm install; npm run build; cd ..`。
3. `copy .env.example .env`，按 [§7](#7-配置项说明) 修改敏感项；端口可用 `HTTP_PORT` / `HTTPS_PORT` 覆盖。

启动 / 运维（PowerShell 或 CMD）：

```powershell
docker compose -f docker-compose.windows.yml up -d --build
docker compose -f docker-compose.windows.yml ps
docker compose -f docker-compose.windows.yml logs -f backend
docker compose -f docker-compose.windows.yml exec backend alembic upgrade head
```

默认访问入口（端口由 `.env` 的 `HTTP_PORT/HTTPS_PORT` 决定）：

| 路径 | 说明 |
| --- | --- |
| `http://localhost:8080/` | 管理后台（Web） |
| `http://localhost:8080/webhook` | TradingView Webhook |
| `ws://<本机IP>:8080/ws/node` | 节点接入 |

Windows 常见问题：

| 现象 | 处理 |
| --- | --- |
| 端口被占用 | `netstat -ano \| findstr :8080` 查占用；或改 `.env` 的 `HTTP_PORT`。要用 80/443 需先确保未被 IIS/HTTP.sys 占用 |
| 绑定挂载失败 | WSL2 后端无需手动共享盘符；Hyper-V 后端需在 Docker Desktop → Settings → Resources → File Sharing 勾选项目所在盘符 |
| nginx 配置解析报错 | 把 `deploy/nginx/default.conf` 转为 LF 换行后 `docker compose -f docker-compose.windows.yml restart nginx` |
| 前端 404 | 是否已执行 `npm run build` 生成 `frontend/dist` |

## 6. 节点端部署（Windows + 真实 MT5）

> 在每台需要跟单的 Windows 主机上执行。一台主机一个 MT5 账户对应一个节点。MT5 账户登录号（`MT5_LOGIN`）是节点的业务唯一键。

1. 安装并登录好 MT5 终端，确保「算法交易/AutoTrading」已开启。
2. 在管理后台 → 「配置」→「节点令牌」复制全局 `NODE_TOKEN`（所有节点共享此令牌）。
3. 将节点客户端（打包 exe，或 `node_client/` 目录）放到 **MT5 安装目录**（与 `terminal64.exe` 同级），配置 `.env`：

```
MANAGER_WS_URL=wss://hub.example.com/ws/node   # 生产用 wss
NODE_TOKEN=<从管理后台复制的全局令牌>
MT5_MOCK=false
WATCH_SYMBOLS=XAUUSD,EURUSD,GBPUSD,USDJPY,US30,US100
```

> 账号：终端已登录则自动复用；未登录时启动交互输入。程序会自动发现同目录的 `terminal64.exe`（或 `terminal.exe`）；找不到会报错退出。

4. 双击 `run_node.bat`（会自动建虚拟环境、装依赖、启动），或手动：

```powershell
cd <MT5安装目录>
python -m venv .venv; .venv\Scripts\activate
pip install -r requirements.txt
pip install MetaTrader5      # 仅 Windows
python node_client.py
```

5. 后台「节点管理 / 总览」确认该节点出现；自动注册默认禁用，需启用后才会上线。若需要修改名称/手数策略/轮询顺序等，在「节点管理 → 编辑」里调整。

> 断线会自动重连（指数退避）；MT5 阻塞调用统一在线程池执行，不影响心跳与上报。重置全局 `NODE_TOKEN` 后，**所有节点的 `.env` 都需同步更新并重启**，否则会被拒绝接入。

## 8. 使用指南（管理后台）

导航栏：**总览** / **节点** / **中控台** / **配置** / **事件**。

### 8.1 中控台（`/console`）

系统级交易规则入口，包含：

- **多区间方向过滤（按品种）**：每个品种一条规则，也是 Webhook 信号的**准入清单**——未登记或取消「启用」的品种一律拒收（含平仓信号；后台手动平仓除外）。
- **全局手数**：开启后，手数策略为 `global` 的节点统一使用该手数。

单品种规则字段说明：

| 字段 | 说明 |
| --- | --- |
| `enabled` | 是否接收并分发该品种信号（`false` 时拒收全部信号，含 Webhook 平仓；后台手动平仓不受影响） |
| `allow_buy` / `allow_sell` | 方向总开关：关闭后该品种不再接收对应方向的信号 |
| `dispatch_mode` | `sync`（全员同步，所有节点并发下单）/ `poll`（轮询轮转，每条信号只由轮到的 1 个节点领取，之后该节点移到队尾循环轮转） |
| `position_scope` | `symbol`（同品种有持仓则拦截）/ `account`（账户有任意持仓即拦截） |
| `default_action` | 价格不在任何区间时：`block` 拦截 / `pass` 放行 |
| `intervals` | 价格区间及每个区间允许的方向 |

示例（`GET/PUT /api/config/filters` 存的就是此结构）：

```json
{
  "XAUUSD": {
    "enabled": true,
    "allow_buy": true,
    "allow_sell": true,
    "dispatch_mode": "sync",
    "position_scope": "symbol",
    "default_action": "block",
    "intervals": [
      { "low": 2300, "high": 2350, "allow": ["BUY"] },
      { "low": 2350, "high": 2400, "allow": ["SELL"] }
    ]
  }
}
```

含义：价格在 2300–2350 只允许 BUY，2350–2400 只允许 SELL，落在区间外按 `default_action` 处理。若 `allow_sell: false`，则无论价格区间如何，所有 SELL 信号都会被拦截。

### 8.2 配置（`/config`）

账户与安全设置，不含交易规则：

- **节点令牌 (NODE_TOKEN)**：查看 / 复制 / 重置全局接入令牌（所有 node_client 共享）。
- **修改密码**、**双因素认证 (2FA)**。

### 8.3 节点管理

- **自动注册（推荐）**：node_client 用任意尚未入库的 `MT5_LOGIN` 首次连接时，后台自动按默认配置入库（名称 `node-<login>`、**禁用**）。**按币种配置**会从中控台已登记品种批量生成（每个品种默认参与 sync/poll、`fixed` + `0.01`、轮询序 0）。入库后须在「节点管理」启用，客户端才会真正接入在线。
- **新建节点（可选）**：在「节点 → + 新建节点」里手动填表。`mt5_login` 全局唯一，重复创建会返回 409。
- **按币种分发配置**：在节点详情或编辑表单中，为每个品种单独设置：
  - `follow_sync` / `follow_poll`：是否参与该品种在对应分发模式下的分发；
  - `lot_mode`：`global`（跟随中控台全局手数）/ `fixed`（本节点该品种固定手数）/ `signal`（跟随信号手数）；
  - `lot`：`fixed` 模式下的手数；
  - `poll_order`：该品种在轮询轮转中的**初始排队顺序**（越小越先轮到；之后按领取情况动态轮转）。
- **品种须双重登记**：信号品种须先在中控台登记；节点 `filters` 中也须存在该品种的按币种条目，否则该节点拒收（事件流 / 事件页会显示中文 `skip_reason`：`节点未配置：…未在节点按币种配置中，拒收`）。
- **编辑/启停**：可随时停用节点（停用后不参与分发，也无法接入）。`mt5_login` 创建后不可修改。

### 8.4 事件（`/events`）

集中查看 Webhook 信号历史（持久化于 MySQL，非仅实时 WS 流）：

- **原始请求体**：每条信号保存 Webhook 收到的 JSON 或纯文本（`raw_payload`），便于对照 TradingView 告警原文。
- **解析字段**：动作、品种、手数、止盈止损、来源 IP、分发模式、信号状态等。
- **节点明细**：展开后可看各节点的下发手数、跳过原因、执行状态、MT5 回报（`retcode` / 订单号 / 成交价 / 错误信息）。
- **分页**：默认每页 20 条，可调整页码与每页条数；不含手动平仓记录。

> 单节点视角的同类数据仍可在「节点详情 → 信号 / 成交回报」Tab 查看；总览页 WS 事件流适合实时监控，事件页适合事后追溯与对账。

### 8.5 监控与远程平仓

- 「总览」实时展示各节点净值、保证金、持仓与盈亏，以及分发/回报事件流。
- 单个持仓可一键平；也可对单节点/全体广播「全平 / 按品种 / 按订单」平仓（全平有二次确认）。

## 9. TradingView Webhook 接入

在 TradingView 警报的「通知 → Webhook URL」填：

```
https://hub.example.com/webhook
```

消息体支持 JSON 或纯文本，示例（JSON）：

```json
{
  "action": "buy",
  "symbol": "XAUUSD",
  "volume": 0.1,
  "sl": 2320.5,
  "tp": 2360.0,
  "token": "<AUTH_TOKEN>"
}
```

- `action` 支持 `buy/sell/close`（含中文「做多/做空/平仓」等关键字）。
- 开启 `ENABLE_AUTH` 后必须带 token，可放在 `token` 字段、`X-Auth-Token` 头或 `?token=`。
- **品种须先在中控台登记且启用**，否则返回 `status: rejected`（见下方示例），不会分发。
- 平仓信号（`close`）会广播给所有在线节点，各自平掉对应品种；须品种已在中控台登记且已启用（CLOSE 不受区间/持仓过滤，但受准入与启用开关约束）。
- `DEDUP_WINDOW` 秒内相同信号（动作/品种/手数/止盈止损一致）会被去重。

未登记品种时的响应示例：

```json
{
  "status": "rejected",
  "signal_id": "sig_...",
  "action": "BUY",
  "symbol": "EURUSD",
  "volume": 0.1,
  "mode": "rejected",
  "targets": 0,
  "reason": "品种未配置：EURUSD未在中控台配置，信号拒收"
}
```

> 字段解析规则与参考仓库 `Robin3910/mt5_python_connector` 完全一致，详见 `backend/app/parser.py` 与 `backend/tests/test_parser.py`。

## 10. 接口与协议速查

### HTTP（均需 `Authorization: Bearer <JWT>`，登录接口除外）

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| POST | `/api/login` | 登录，返回 JWT |
| GET/POST | `/api/nodes` | 列表 / 新建节点（`mt5_login` 重复返回 409）|
| GET/PATCH/DELETE | `/api/nodes/{id}` | 查询 / 更新 / 删除 |
| GET | `/api/config/node-token` | 获取全局节点接入令牌（明文） |
| POST | `/api/config/node-token/rotate` | 重置全局节点接入令牌（旧令牌立即失效）|
| GET/PUT | `/api/config/lot` | 全局手数 |
| GET/PUT | `/api/config/filters` | 按品种规则（准入清单 + 方向总开关 + 分发模式/持仓范围 + 区间过滤） |
| GET | `/api/events/signals` | Webhook 信号事件分页（`page`/`page_size`；含 `raw_payload` 与各节点 `dispatches`） |
| GET | `/api/accounts` | 全部账户快照 |
| POST | `/api/nodes/{id}/close` · `/api/close-all` · `/api/close-batch` | 单节点 / 全局 / 多节点平仓 |
| POST | `/webhook` | TradingView 信号入口 |

### WebSocket

- `/ws/node`：节点接入。**首包必须**为 `{"type":"auth","data":{"token":"<NODE_TOKEN>","mt5_login":<int>}}`，后台依次校验：① `token` 是否与全局 `NODE_TOKEN` 一致（常量时间比较）② `mt5_login` 是否对应已注册节点（未注册则自动入库）③ 节点是否启用 ④ 是否重复在线。通过即回 `auth_ok` 并开始心跳/上报；命令 `open`/`close` 下行，结果 `trade_result` 上行。鉴权失败的关闭码：`4400` 首包超时/格式错误、`4401` 鉴权失败（`invalid_token`/`missing_mt5_login`）、`4403` 节点被禁用、`4409` 重复在线。
- `/ws/admin?token=<JWT>`：后台实时推送 `snapshot/node_status/node_registered/node_rejected/account/dispatch/trade_result`（其中 `node_registered` 为自动注册事件）。

## 12. 故障排查（FAQ）

| 现象 | 排查方向 |
| --- | --- |
| 节点一直离线 | 全局 `NODE_TOKEN` 是否与「配置 → 节点令牌」一致（被重置过吗？）；`MANAGER_WS_URL` 是否可达；Nginx 是否放行 `/ws/`（已配置 Upgrade）|
| 节点被拒绝（`invalid_token`）| 全局令牌已变更，重新从「配置 → 节点令牌」复制并更新该节点 `.env` 后重启 |
| 节点被拒绝（`missing_mt5_login`）| `.env` 缺少 `MT5_LOGIN` 或值不为正整数 |
| 节点被拒绝（`disabled`）| 节点已在管理后台被禁用，到「节点管理」启用即可 |
| 节点被拒绝（`already_online`）| 同一 `MT5_LOGIN` 已有在线连接（仅允许一个同时在线）；等旧连接超时或显式下线 |
| Webhook 返回 401 | `ENABLE_AUTH=true` 时是否带对 `AUTH_TOKEN` |
| Webhook 返回 403 | IP 白名单是否开启；经 Nginx 时是否正确透传 `X-Forwarded-For` |
| Webhook 返回 400 | 信号无法解析：检查 `action`/`symbol` 是否齐全（参考 §9 示例） |
| Webhook 返回 `status: rejected` | 品种未在中控台登记；到「中控台」添加该品种并保存 |
| 信号被「跳过」 | 命中方向总开关、持仓过滤、区间过滤，或**节点未配该品种**（事件页 / 节点详情会显示中文 `skip_reason`） |
| 下单报 `10030` | 填充模式不支持——节点端已自动切换 FOK/IOC/RETURN，仍失败请检查券商品种设置 |
| 节点端 `MetaTrader5 not available` | 该包仅 Windows；非 Windows 调试请设 `MT5_MOCK=true` |
| 后端起不来连不上 DB | 生产确认 `MYSQL_DSN`、容器 `mysql` 健康；本地默认 SQLite 无需 DB |
| Redis 不可用 | 后端会降级启动并告警，但实时态/队列/锁不可用，请尽快恢复 Redis |

查看日志：`docker compose logs -f backend`、节点端控制台输出、Nginx 容器日志。
